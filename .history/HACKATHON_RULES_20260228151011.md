## Eligibility

- **Open to:** Individuals 18+, teams, and organizations (excluding Brazil, Quebec, Russia, Crimea, Cuba, Iran, North Korea, Syria, and OFAC-designated countries).
- **Not open to:** Promotion entities, their employees/agents/family, judges or their employers, and affiliates.
- **Submitter types:** **Student Developer** (enrolled / learning) or **Professional Developer** (everyone else). All team members must match the chosen type for prize eligibility.

---

## Project requirements

**What to build:** A generative AI application using **Amazon Nova on AWS**. Core solution must use at least one of: Nova foundation model (Nova 2 Lite, Nova 2 Sonic, multimodal embedding) and/or **Nova Act**.

**Categories (pick one):**

1. **Agentic AI** — Agents using Nova reasoning for complex, real-world problems.
2. **Multimodal Understanding** — Text, documents, speech, image, or video.
3. **UI Automation** — Nova Act–powered agents automating web UI workflows.
4. **Voice AI** — Real-time conversational voice with Nova 2 Sonic.
5. **Freestyle** — Other Nova-based projects.

**Other:** You may use preferred services/frameworks (e.g. AWS, Langchain, MCP, A2A). Project must run on its intended platform, be installable and functional, and be new or meaningfully updated during the submission period. Third-party SDKs/APIs must be used with proper authorization.

---

## Submission requirements

- **Project** built with required tools and meeting the project requirements above.
- **Submitter type:** Student Developer Team **or** Professional Developer Team.
- **One category** selected for the project.
- **Text description:** Brief summary, purpose, and how it uses Amazon Nova.
- **Demo video** (~3 minutes):
  - Shows the project working.
  - Hosted and public on **YouTube, Vimeo, or Youku**; link on submission form.
  - Include hashtag **#AmazonNova**.
  - No unauthorized third-party trademarks or copyrighted music.
- **Code repository:** If private, share with **testing@devpost.com** and **Amazon-Nova-hackathon@amazon.com**.
- **Testing access:** Working project available for judges (e.g. live site, demo, or test build). Free access; include login credentials if needed. Keep available until the judging period ends.
- **Language:** All materials in English (or provide English translation for video, description, and testing instructions).

---

## Judging

**Stage 1:** Pass/fail — project fits theme and uses required APIs/SDKs.

**Stage 2 (Judging criteria):**

| Criterion | Weight |
|-----------|--------|
| **Technical Implementation** — Quality, effectiveness, Nova integration, architecture | **60%** |
| **Enterprise or Community Impact** — Business value or community benefits | **20%** |
| **Creativity and Innovation** — Novelty, innovative use of multi-agent systems | **20%** |

Ties: highest score on the first criterion wins; if still tied, repeat with next criterion, then judge vote.
